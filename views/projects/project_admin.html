Admin access to projects

This page allows administrators to see all of the project details and edit details that are locked down for supervisors. As well as linking to the project details, the table also shows the current availability of each project and provides actions to conceal or readvertise projects: click on the tabs below for details.

  • Follow {{=A('this link', _href=URL('projects', 'project_details', scheme=True, host=True))}} to create a completely new proposal.
  • You can edit the details of existing proposals by clicking on the project title in the table below.
  • The project detail page is also used to accept a student on to a project.
All projects from previous academic year are automatically hidden from students.
Unallocated current projects are visible to students.
Allocated current projects are also visible to students.
Unallocated current projects can also be concealed (see project actions tab).
Clicking on this action icon will conceal an unallocated current project from students. This should be used to withdraw projects that become impossible to support, for example due to equipment failure or travel restrictions, and not to manage student numbers. We want the proposal database to showcase the full range of research available through the courses to current and future students.
Clicking on this action icon readvertises a project. If a project has already been allocated to current or past student, the project details are duplicated to keep the original project pairing and make it available to new students. Otherwise, the project is simply updated to make it available to the current academic year.
